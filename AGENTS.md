# AGENTS.md ‚Äî NixOS flake

## Rebuild

```sh
sudo nixos-rebuild switch --flake '.#HOST'   # HOST = folder name under system
nix-collect-garbage -d                        # user gc
sudo /run/current-system/bin/switch-to-configuration boot  # boot after gc
```

## Host system layout

```
system/<hostname>/
‚îú‚îÄ‚îÄ config.nix         # { dpi, system, kbLayout, bootPath } - consumed by flake.nix
‚îú‚îÄ‚îÄ hardware.nix       # generated by nixos-generate-config (do not edit manually)
‚îî‚îÄ‚îÄ configuration.nix  # per-host config
```

Host names: `pc`, `t14`, `xps15`. Each is auto-discovered from `system/` subdirectory names.

## Secrets

- `secrets/*.json` are **git-crypt encrypted** (`.gitattributes` defines the filter).
- Loaded in `flake.nix` via `builtins.fromJSON` ‚Äî each file becomes a top-level key.
- Contains `user.json` (name + password hash). Username is `binh`.
- An agent cannot read them without `git-crypt unlock`.

## Structure

- **`configuration.nix`** ‚Äî system entry point; auto-imports all `core/*.nix`
- **`core/`** ‚Äî shared system modules (font, session, system, user)
- **`modules/`** ‚Äî optional feature modules: `8bitdo`, `game`, `nordvpn`, `qudelix`, `uutils`
- **`overlays/`** ‚Äî auto-imported overlay *.nix files (chromium, common, rofi)
- **`home/`** ‚Äî home-manager config (shared across hosts via `home/home.nix`); each file is a module
- **`system/<hostname>/`** ‚Äî per-host config as described above

## Pi coding-agent (split architecture)

The `pi` coding-agent (from `github:lukasl-dev/pi.nix`) is split across two layers:

| Layer | Binary | Source | PATH priority |
|-------|--------|--------|---------------|
| System | Raw `pi` binary | `environment.systemPackages` via `modules/pi.nix` (NixOS module) | `/run/current-system/sw/bin/pi` |
| User | Wrapped `pi` with jq-merge for settings | `home.packages` via `home/pi.nix` (HM module) | `~/.nix-profile/bin/pi` (takes priority) |

- **System-level install** (`modules/pi.nix`): Imports `pi-nix.nixosModules.default` and enables `programs.pi.coding-agent`. Makes the binary available to all users and system scripts.
- **User-level config** (`home/pi.nix`): Imports `pi-nix.homeModules.default` and configures settings (packages list, rules, skills, extensions). The HM binary shadows the system one in the user's PATH.

### Rebuild

```sh
sudo nixos-rebuild switch --flake '.#HOST'   # re-evaluates both system and HM pi modules
```

## Desktop / display

- SDDM with Wayland, auto-login as `binh`, default session `mango`
- Mango is the compositor. DMS (DankMaterialShell) provides widgets (system monitor, VPN, audio visualizer, calendar, clipboard via `wtype`).
- Fcitx5 with Unikey for Vietnamese input.

## Noteworthy config

- `stateVersion = "25.11"` (set in `flake.nix`)
- `nix.settings.experimental-features = ["nix-command" "flakes"]` (in `configuration.nix`)
- Firewall disabled (`networking.firewall.enable = false`)
- NextDNS over TLS via `services.resolved`
- Docker data root at `/docker`
- IWD WiFi backend
- `uutils` module replaces coreutils/findutils/diffutils with Rust equivalents
